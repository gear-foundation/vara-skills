# Sails Ethexe Patterns

Source of truth: `rs/macros/core/src/service/ethexe.rs`, `rs/src/gstd/ethexe.rs`, `rs/src/solidity.rs`, `rs/sol-gen/`.

## Feature Flag

Enable with `sails-rs = { version = "...", features = ["ethexe"] }`. This pulls in:
- `alloy-primitives` (B256, Selector)
- `alloy-sol-types` (SolValue, SolType, ABI encoding)
- Propagates through macros, type-registry, and reflect-hash

## Standard Sails vs Ethexe Diff

| Area | Standard Sails | Ethexe |
|------|---------------|--------|
| Type encoding | SCALE codec | ABI (`SolValue`) for ethabi transport, SCALE for scale transport |
| Event emission | `emit_event(MyEvent::Variant)` | `emit_eth_event(MyEvent::Variant)` → sends to `ETH_EVENT_ADDR` |
| Event traits | `SailsEvent` only | `SailsEvent` + `EthEvent` (both auto-generated by `#[event]`) |
| Transport | Scale only | Scale + ethabi (dual or single via `#[export]` flags) |
| Payable | Not available | `#[export(payable)]` or `#[export(ethabi, payable)]` |
| Gated syscalls | Full set | `signal_from`, `signal_code`, `system_reserve_gas` unavailable |
| Program entry | Sails Header routing | Sails Header + Solidity selector routing (`sol_init`, `sol_main`) |
| Client gen | `sails-client-gen-v2` | + `sails-sol-gen` for Solidity interface generation |
| Metadata | Route/entry IDs | + Solidity method/constructor/callback signatures |

## Type Requirements

Types used in ethabi-exported methods must implement `SolValue` (from `alloy-sol-types`). Primitive types and tuples implement it automatically. For custom structs/enums:

- `#[sails_type]` handles standard Sails derives (Encode, Decode, TypeInfo, ReflectHash)
- Ethabi transport additionally requires the type to be ABI-encodable as a Solidity tuple
- Parameters decode via `SolValue::abi_decode_params()`, encode via `abi_encode_sequence()`
- Message IDs are encoded as `alloy_primitives::B256`

## Transport Selection

```rust
#[export]                        // Both: scale + ethabi (default when ethexe on)
#[export(scale)]                 // Scale only — no ethabi even with ethexe
#[export(ethabi)]                // Ethabi only
#[export(scale, ethabi)]         // Explicit dual
#[export(ethabi, payable)]       // Ethabi + payable
#[export(scale, ethabi, payable)] // Dual + payable
```

- If no transport flag: both enabled
- If any flag specified: only those listed
- `payable` requires `ethabi` — error if scale-only

## Payable Methods

```rust
#[export(payable)]
pub fn do_something_payable(&mut self, p1: u32) -> u32 { ... }
```

Runtime check: non-payable methods panic if `msg::value() > 0`. Payable methods skip this check.

Constructors can also be payable:
```rust
#[export(payable)]
pub fn create_payable() -> Self { ... }
```

## Event Emission

### Standard Gear events
```rust
self.emit_event(MyEvent::Something { value: 42 });
```

### Eth events (ethexe only)
```rust
self.emit_eth_event(MyEvent::Something { value: 42 });
```

- Sends to `ETH_EVENT_ADDR` (`0xffffffffffffffffffffffffffffffffffffffff`)
- Encodes topics (keccak256 signature + indexed fields) + ABI-encoded data
- Max 3 `#[indexed]` fields per event variant

### Event Trait Generation

`#[event]` macro generates both `SailsEvent` and `EthEvent` impls when ethexe is enabled:
```rust
#[event]
pub enum MyEvent {
    Something {
        #[indexed]
        who: ActorId,
        value: u64,
    },
    Simple,
}
```

**Forbidden event variant names** (reserved by runtime):
ExecutableBalanceTopUpRequested, Message, MessageQueueingRequested, Reply, ReplyQueueingRequested, StateChanged, ValueClaimed, ValueClaimingRequested

## Solidity Interface Generation

`sails-sol-gen` generates Solidity contract interfaces from IDL:
- Function stubs with correct ABI signatures
- Event definitions with `indexed` fields
- Constructor interfaces
- `payable` annotation on applicable functions

Usage: `generate_solidity_contract(idl_content, name)` → Solidity source

## Program Entry Points

Ethexe programs generate additional entry points:
- `sol_init()` — handles constructor calls via Solidity selector routing
- `sol_main()` — handles service method calls via selector routing

Selector computation: `keccak256(service_name + method_name + param_types)` → 4-byte selector

Callback signatures: `keccak256("replyOn_" + method_name + return_types)`

## Gated Syscalls

Available in ethexe: all `Syscall::*` methods except:
- `Syscall::signal_from()` — `#[cfg(not(feature = "ethexe"))]`
- `Syscall::signal_code()` — `#[cfg(not(feature = "ethexe"))]`
- `Syscall::system_reserve_gas()` — `#[cfg(not(feature = "ethexe"))]`

## Ethexe Workspace Structure

Separate workspace at `rs/ethexe/` with its own `Cargo.toml`. Example project layout:

```
rs/ethexe/
├── Cargo.toml              # Workspace root
├── ethapp/                 # Minimal ethexe program
│   ├── Cargo.toml          # sails-rs with features = ["ethexe"]
│   ├── build.rs
│   └── src/lib.rs
├── ethapp_with_events/     # Program with eth event emission
└── macros-tests/           # Macro snapshot tests
```

Dependencies pattern:
```toml
[dependencies]
sails-rs = { workspace = true, features = ["ethexe"] }

[build-dependencies]
sails-rs = { workspace = true, features = ["ethexe", "wasm-builder"] }

[dev-dependencies]
sails-rs = { workspace = true, features = ["ethexe", "gtest"] }
```

## Validation Rules

- Service names validated against Solidity reserved keywords
- PascalCase service names required
- `#[indexed]` fields: max 3 per event variant
- Forbidden event variant names (see above)
